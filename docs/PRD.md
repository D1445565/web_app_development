# 專案名稱：食譜收藏夾

## 1. 專案概述
- **背景與動機**：現代人生活忙碌，有時想下廚卻不知道做什麼，或是想要尋找新的料理靈感；而對於有經驗的廚師或料理愛好者，也需要一個平台來有系統地記錄和分享自己的拿手好菜。因此需要一個可以方便記錄、搜尋與收藏食譜的網站。
- **目標用戶**：一般人、廚師
- **核心價值主張**：提供一個直覺、易用的食譜管理與查詢平台，讓使用者能輕鬆地記錄、尋找及收藏各種美味料理的作法，提升下廚的樂趣與便利性。

## 2. 功能需求

以下列出系統的主要功能及對應的使用者故事：

1. **新增食譜**
   - **使用者故事**：作為 **廚師或一般人**，我希望 **能夠新增包含名稱、簡介及圖片的食譜**，以便 **記錄我的私房菜或新研發的料理**。
2. **列出材料與步驟**
   - **使用者故事**：作為 **廚師或一般人**，我希望 **在食譜頁面中能詳細編輯與檢視所需的材料清單和圖文料理步驟**，以便 **讓自己或其他人能精準地跟著做**。
3. **搜尋食譜**
   - **使用者故事**：作為 **一般人**，我希望 **能夠透過關鍵字或食材名稱搜尋食譜**，以便 **快速找到符合我目前需求或冰箱剩餘食材的料理**。
4. **食譜分類與標籤**
   - **使用者故事**：作為 **一般人**，我希望 **能夠瀏覽不同分類或標籤的食譜（例如：中式、西式、甜點、快速料理、素食等）**，以便 **在我不知道要吃什麼時獲得靈感**。
5. **收藏食譜**
   - **使用者故事**：作為 **一般人**，我希望 **能夠將喜歡的食譜加入個人的「收藏夾」中**，以便 **未來想做這道菜時可以快速找出來，不用重新搜尋**。

## 3. 非功能需求
- **技術限制**：
  - 預計使用 Flask 框架。
  - 搭配 Jinja2 作為前端模板引擎。
  - 使用 SQLite 作為關聯式資料庫。
- **效能與安全考量**：
  - 確保搜尋功能的響應時間在合理範圍內。
  - 實作防護機制，防止 SQL Injection 及 XSS 攻擊，確保使用者輸入的內容安全。
  - (若有會員功能) 提供基礎的密碼加密儲存與安全驗證機制。

## 4. MVP 範圍
- **Must Have (必須有)**：
  - 瀏覽所有食譜列表
  - 新增食譜（包含名稱、材料、步驟）
  - 檢視單一食譜的詳細內容
  - 基本的關鍵字搜尋功能
- **Should Have (應該有)**：
  - 使用者註冊與登入功能
  - 將食譜加入我的收藏夾
  - 編輯與刪除自己建立的食譜
- **Nice to Have (可以有)**：
  - 食譜分類與標籤篩選
  - 食譜圖片上傳功能
  - 食譜評分與留言互動
